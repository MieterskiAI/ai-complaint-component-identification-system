# Operator Checklist (before final decision)

## Data integrity
- [ ] Complaint ID and Order ID match the same transaction.
- [ ] Product SKU is valid and exists in the compatibility list.
- [ ] Dates are consistent (purchase ≤ delivery ≤ complaint).

## Symptom validation
- [ ] Symptoms are specific and not contradictory.
- [ ] Attachments (if any) align with the described issue.
- [ ] Environmental context (usage, conditions) is present.

## Compatibility verification
- [ ] Eliminated components are clearly incompatible with the SKU/variant.
- [ ] Top candidates are compatible with the exact SKU revision.
- [ ] Any mandatory accessory/kit requirements are noted.

## Risk & escalation
- [ ] Safety-related issues are flagged for escalation.
- [ ] Warranty status is confirmed (or set to follow-up).
- [ ] If uncertain, request additional info before dispatch.

## Decision logging
- [ ] Final decision includes reasoning and evidence.
- [ ] Decision is written back to the complaint record.
